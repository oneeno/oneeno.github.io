读《程序员修炼手册》

一、学会观察这个世界

当你走向社会的时候，不要太过于相信他人（包括网络上的软文，机构等）的说法。不要把自己当成学生或者说自己还是个孩子，不要把自己的将来交给别人，学会用自己的眼球去观察，去了解这个世界。

网络上的一些所谓的新闻，或者鸡汤文有可能是枪手写的，不必太过较真。在学英语方法，有国外背景的教程或者课程，未必真正适合自己，适合自己的才是最好的。

如果不知道自己未来的方向，可以通过一些招聘网站上查看相关数据。这些数据都是真实不掺假的原始数据。每隔一段时间搜索一次，从中寻找一些规律，然后总结出来。

比如，我在http://www.jobui.com中搜索Python，全国大概有7万个职位，然后多翻几页，这样不仅能看清楚大概有多少是有效的职位，而且还可以看清楚用人单位最关心的技能是什么，这个很重要，决定你将来要花多少时间和精力在特定的知识点上，比如说 numpy，搜了下，全国只有500个职位，仔细读一下，也并非全是有效职位，花太多时间和精力在这个上面就不值得了。当然，自己感兴趣或者为了研究的话，另当别论。但是如果是为了快速就业，把太多时间和精力，花在很多职业生涯前一两年用不到的知识上就太可惜了。同时，并非某一个行业的职位数目少，就不能当作一个方向进行选择，比如PHP，我搜索到的职位数是5000左右，比 Python 少，是不是就选 Python了呢？我的回答是：不一定，因为 Python 职位虽然多，但是可能 竞争者也多，有些方向虽然职位数不多，但是和你竞争这些职位的人数量也少，不见得就不能选择。当然，最终还是要根据自己的兴趣来选择，这样才有动力，不然，学到一半就放弃了，白白浪费时间和精力。

二、学会学习，再去学习

虽然 我们每天都在学习，但是80%的人事实上并不会学习，所以，我们要“先学会如何学习，再开始学习”，才能事半功倍。

学习的客观规律是“守破离”

1，守

刚开始学习，首先要做到对老师和知识的“敬畏之心”。学习就要先信服老师，不服老师就别学，你服谁，就去学谁，不服，不如不去，谁都不服，那你已经天下第一了，不用学了。

批判，挑剔很容易，但这不是学习的态度。我们可能经常会挑刺，老师这个毛病那个毛病，老师也是普通人，不是圣人，谁还没有缺点呢？我们要关注的是，从老师那里能学到什么，得到什么，老师身上有哪些地方是值得你学习和利用的。这才是正确的学习思维。

注意：批判别人，并不能使自己进步！

所以，刚开始，你要照着老师说的，不打一点折扣的去执行，在练习的过程中慢慢体会，才能达到下一个“破”的境界，就像武侠小说里，只有牢牢记住每个招式，才能灵活应对。

“守” 就是一心遵守老师教诲，守住招式，反复练习，这才是最基本，也是最重要的也阶段。

2，破

一招一式 “守” 好了，自然就到了 “破” 的阶段，也就是在试错过程中发挥自己的独创。开始“突破”老师的教诲，开始有自己的思考，自己的思考 多了，自然到了“离”的阶段。

3，离

“离”就是通过不断的思考和反复的练习，一点点“破”，最后形成了自己的风格。“脱离”老师开始讲的“招式”，升华成为“专家”。

很多人“守”都没守好，就想着“脱离老师”，结果自我意淫，自欺欺人。“追新逐异，自我表现“这是人性的弱点。

三、多动手，多实战

无论如何，请坚持不懈的动手实战！

无论是学习绘画，还是学习期货任何技术，一定要动手，仅仅靠看和听是学不好的。

就像写作文，要有自己的思路，一般来说要经过以下几个阶段。首先学习词汇，其次学习造句，然后大量阅读别人的文章，自己模仿着写一些，逐渐积累经验，才能形成自己的思路。

学编程，也是同样的道理，只是看一看，听一听视频，是不足以学好编程的。

所以，多动手，跟着做，一行一行的跟着敲，一个项目敲完了，再试卷加一些自己的功能，按照自己的思路敲一些代码，收获远比于听大得多。

如果遇到一些不能理解的，那么先动手写，先写代码，量变会引起质变的，而这种质变的确发生过不少次。

重点提醒：要先理解代码思路之后，再跟着敲，千万不要左边摆着别人的程序，右边自己一个个字的写，这就不再是程序员了，成了打字员。

所以，别问那么多，别看那么多，别想那么多，开始动手写吧。

四、建立体系为先，不纠结，不追求完美。

对待人生的任何事情都要：抓大放小，要事为先

任何事情 都要追求完美才敢继续往后进行，是一种性格缺陷。大胆的我放弃一些东西，有失才有得，把自己有限的，宝贵的精力用在对提升自己直接相关的地方，这才是有效的学习方式！

等你参加工作了，有了可持续发展的事业动力和经济基础，有时间有精力闲着没事干的时候，再去研究那些其它的。

有这样一个关于产能与产量关系的故事。

一个人喜欢读书，读了一辈子，无所不通。这时候要做任何工作都可以，他的产能到了极致。但是，他老了，即使每天产出大量的东西也不能维持多久了，一辈子的产量并没有多少。

另一个人直接工作，天天在机器旁日复一日，做了一辈子，忙，没时间学习和进修，可以想象，他的产量又有多大呢？

人的一生，要锻炼产能，也要及时产出，引擎要转，也要停下来加油，这两者平衡得越好，你的贡献和产量就越大。如果钻研的东西和产能目标（或者说近期的产能目标）无关的话，要舍得放手。

所以，对于编辑中各种协议，各种过于详细的细节，各种华而不实的效果，都暂时和它们”分手“吧，一切和我们的直接工作目标关联不大的东西，扔在一边或者弄清楚到足够支持下一步学习就可以了。

五、什么是高手？

你和高手的差距不在于某个小知识点，而在于体系。高手往往掌握了”成系统的知识体系“，应用这套”系统知识“可以解决他遇到的绝大多数问题。

所以，要想成为高手，先建立”知识体系“，再根据需要去钻石某些细节。

我一向不认为一个人掌握了很多很多知识点的细节就是高手。一个人如果告诉你，”回“字有四种写法。你会认为他是高手吗？毫不夸张的说，现在你所想学的所有知识细节都在网上或书中都可以找到，只要你肯花时间花精力去动手试，去钻研，没有什么是不能掌握的。比如：java 中的 servlet，spring 你能掌握吗，当然能？但是如果你把时间花在研究spring 的各种细节上，光一个API文档就够费时的，这样就太浪费精力了。而且学习和积极性会受到打击，谁愿意整天泡在细节的蜘蛛网中挣扎不出来呢？谁愿意天天经历风雨却总是看不到彩虹呢？

盖房子，要先建骨架，再谈装修。

画山水，要先画结构，再谈润色。

学习要学会变通。要在合适的时间，做合适的事情！

把时间和精力花在做项目上面，花在写作品以及锻炼解决问题的能力上面。这是迈向高手的正确途径，而且是快速有效的方法。

当你看书到某个地方暂时不理解时，暂时放手，追求一些行云流水，自然而然的境界。只是不要停下前进的脚本，不要被大路旁边的细枝末节干扰了你前进的行程。项目，真实的项目，这才是目的。

陶渊明读书，叫做“不求甚解”，但也大诗人。

诸葛亮读书，总是“观其大略”，但也是大政治家。

作研究时，先确定了大方向之后，再详细深入研究。

作为应用型人才，是研究“回”字的n种写法，还是抓紧时间向着主要应用的方向努力，你需要作出抉择。

以项目实战驱动去学习，当把握了技术的脉络之后，再去补充研究细节，才是正途。

六、问题解决之道（会提问，会搜索）

既然是学习，就不可能不会遇到问题。

既然是自觉，就没有和老师探讨机会。

那么在自觉途中遇到一些恼人的问题怎么办。

你遇到的每一个问题都是涨经验的机会，每解决一个问题，你的经验值就会上升几百点，问题遇到得越多，知识提升得就越快。

1，当遇到问题时，要仔细观察错误的现象，不要随便问人！

在开发中，仔细观察出错信息，或者运行不正常的信息，是你要做的第一件事，读清楚了，才能在以后的步骤中有的放矢。

这里分三种情况。

  1. 错误信息读懂了，那么请进入 第2步，要仔细思考问题会出在哪些环节。
  2. 没读懂，愣是一点没看懂，进入第5步吧，百度。
  3. 读了个半懂，有些眉目，但是不太能确定，第2步和第4步结合来看。

2，要仔细思考问题会出在哪些环节（重要）

当你读懂了一个问题之后，要好好思考这个问题可能会在哪些环节上出错。

3，如何定位错误？分析清楚有哪些环节之后，下一步就是定位到底什么环节出错了。

定位有三种方法：

  • A，简单打印输出，可以解决一些简单问题，比如 Python 的print()。这种方法常用，必须掌握。
  • B，Debug，通过调试查看程度运行细节，解决复杂问题。
  • 以，删掉一部分高度一部分，也就是去掉一部分的功能，做减法，做简化，然后再高度剩下的功能，JavaScripti常用。

4，如果还不行，Baidu或者Google

如果以上方法你还查不出来，那么你遇到的错误是值得认真对待的。是会影响你学习生涯的错误，问一下Google或者百度，照着下面的方法查查看。

  • A，系统化知识
  • B，真实项目锻炼经验
  • C，少走弯路，节省时间
  • D，形成学习圈子（如找相应的群等）

如何查搜索引擎？

先精后粗，首先尽量精确的查找

比如有一个错误，Socket，你怀疑它是在 connect() 方法出的问题，那么当然是选这样的关键词

先中后英，本着以解决问题为主的想法，练习英文还是先放一边。

首先应该在中文网页中查询，还不行的话，搜索英文吧。

阅读英文书箱就像隔在你面前的窗户纸，远看灰蒙蒙怪唬人的，你壮着胆子一捅，它就破了。宁可在沙场上战死，也不能被吓死。

信息筛选

搜索出来的结果不见得能够完全匹配，建议是多阅读前几页的搜索结果，多打开几个网页看看。一般而言，超过3页就没意义了，如果超过3页还没找到合适的答案，或许你应该调整一下关键词，或者放粗整个搜索的结果了。

经常进行知识难点的查询。如果一个问题牵扯的面比较广，就干脆到网上搜索一些相关的专题，比如 Python乱码mysql之类的，如果有必要，不要犯懒，动手写一些小的测试程序，来弄明白知识点的细节，这是涨知识的重要途径。

5，如果还不行，那么就去 BBS和QQ群

如果以上方法都试了还是不行，那么，就到论坛或QQ群上面问一问高手吧。向别人提问是非常需要的技巧。

向别人提问，首先确定你已经做了自己应该做的事，简单的说就是前面的1，2，3，4步你都做过了，然后再求助于人，不要没有经过认真思考就草率的向别人提问，自己也不会有长足进步的。

6，还解决不了

这时就应该考虑是不是思考从根上就搞错了。
